## 3. Work Performed
Implementations and updates from this codebase update.

### 3.1 New FlySky Drive Stack Added (`Wheel/BLDC_Odrive/FlySky/`)
- Added `esp32_flysky.ino`:
  - Reads FlySky PWM channels (CH1/CH2) using interrupt-based pulse timing.
  - Streams channel values over serial in a simple text protocol.
- Added `flysky.py`:
  - Direct serial PWM-to-ODrive differential drive bridge.
  - Includes deadband handling, signal-loss failsafe, velocity limiting, and ODrive reconnect logic.
- Added `odrive_cmd_vel.py`:
  - ROS2 node subscribing to `TwistStamped` and converting to left/right wheel turns-per-second.
  - Applies track width/wheel diameter kinematics and right-side motor inversion.
- Added `wasd_teleop.py`:
  - Keyboard teleop ROS2 publisher for `rover_controller/cmd_vel`.
  - Includes hold-based motion behavior and deadman timeout handling.
- Added `README.md`:
  - Documents end-to-end FlySky -> serial -> ODrive architecture.
  - Captures hardware mapping, tuning notes, service behavior, and deployment commands.

### 3.2 WebSocket ROS Drive Controller Updated (`ws_to_diffdrive_ros.py`)
- Removed watchdog timeout configuration constants and runtime watchdog task.
- Removed watchdog heartbeat and failsafe-state tracking logic.
- Updated startup logs to explicitly note watchdog disabled state.
- Kept core command handling, motor actuation, and telemetry broadcasting flow intact.
