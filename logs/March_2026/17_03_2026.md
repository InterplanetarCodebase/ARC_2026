## 3. Work Performed
Implementations from today.

- Image capture feature added in camera receiver (`Capture` button), saving to `Images/{camera_id}_{timestamp}.jpg`.
- QR code scanning added via `Scan` button and external scanner script invocation.
- ArUco tag scanning added in scanner using OpenCV predefined dictionaries, returning dictionary name and marker ID.
- Scanner flow aligned to `scan.py` with GUI output for decoded QR/ArUco results.
- Receiver tile controls refined (top-center controls, `Scan` + `Capture` layout and naming updates).
- Receiver `Settings` popover added to control per-camera runtime parameters:
	- `enabled` on/off
	- `device` (`/dev/videoX`)
	- `port`
	- `bitrate`
	- `width` / `height`
	- `fps`
- Receiver now sends runtime control commands to transmitter over UDP JSON (`set_camera` command).
- Receiver now locally reconfigures matching UDP receive backend port after settings apply.
- Transmitter runtime control listener added to apply per-camera changes without full process restart.
- Dynamic camera slot creation implemented in transmitter so camera IDs can be started from receiver settings when transmitter is launched without `-c`.
- Runtime reliability fixes completed:
	- fixed settings button/popover activation issue in GTK
	- fixed control-path deadlock during apply/start sequence
	- adjusted retry policy to avoid redundant retries for control-triggered failures
	- kept monitor restart path for unexpected runtime disconnect/failure cases
	- added native format/io-mode reprobe during control apply to prevent parse/link failures on dynamic starts
- New default startup profile set:
	- Receiver defaults to 8 ports: `5000` to `5007`
	- Receiver default decode resolution: `1920x1080`
	- Receiver default camera settings: `bitrate=1000000` (1 Mbps), `fps=25`
	- Transmitter CLI defaults aligned to `bitrate=1000000`, `fps=25`
